# Spec：结局分享卡

> 在所有结局画面（legendary / excellent / profitable / average / bad_release / counter_win / lose）的文案下方、「再来一局」按钮上方，插入一张分享卡。
> 技术方案：纯 CSS 样式卡片，玩家截图分享。无需 canvas / html2canvas。

---

## 一、卡片内容

### 1.1 固定信息区

```
游戏名（小字）：完蛋！我被延期包围了
结局等级（大字 + emoji + 对应颜色）
方向 · 年份（如：开放世界 · 2019年，方向未定则不显示）
存活周数（state.survived）
```

### 1.2 隐性评价条

五个维度，每个显示名称 + 色块条 + 数字（0-10）：

| 字段 | 显示名 |
|------|--------|
| honesty | 诚实 |
| people | 人情 |
| quality | 品质 |
| judgment | 判断 |
| grit | 抗压 |

色块条用 emoji 或简单字符模拟（`█` 填充，`░` 空白），宽度固定 10 格：

```
诚实   ██████░░░░  6
```

### 1.3 评语（verdict）

根据五维分布生成一句话，带引号，置于卡片底部。

**评语优先级逻辑**（从上到下，命中即用）：

| 条件 | 评语 |
|------|------|
| honesty ≥ 7 且 quality ≥ 7 | 「你做对了，但这个行业不一定记得你。」 |
| honesty ≥ 7 且 quality < 5 | 「你很诚实。关于品质这件事，你也从没骗过自己。」 |
| honesty < 4 且 gamePhase 在 legendary / excellent | 「你赢了。用了你自己才知道的方式。」 |
| honesty < 4 且 gamePhase 在 lose / bad_release | 「数字是编出来的，结局是真实的。」 |
| judgment ≥ 8 且 people < 4 | 「你看得准，但你不在乎那些人。他们也不会记得你。」 |
| people ≥ 8 且 judgment < 4 | 「你在乎所有人，但你看不准。代价高一些而已。」 |
| grit ≥ 8 且 gamePhase 在 lose | 「你撑到了最后。只是最后还不够。」 |
| grit < 4 且 gamePhase 在 legendary / excellent | 「你赢了，而且你自己都有点意外。」 |
| 全部 ≥ 7（五项总分 ≥ 35） | 「你做到了几乎所有事情。你知道这有多难。」 |
| 全部 ≤ 4（五项总分 ≤ 20） | 「有时候这个行业就是这样。不全是你的问题。」 |
| 兜底 | 「这一局是你的。好与坏，都是你的。」 |

### 1.4 提示文字

卡片最底部小字：`截图，分享给你的制作人朋友`

## 二、卡片样式

```
背景色：#0c0c18（比游戏主背景略浅，与页面区分）
边框：1px solid #2a2a3a，圆角 12px
内边距：20px
最大宽度：340px，水平居中
字体：monospace
```

结局等级颜色沿用现有：
- legendary：`#facc15`（金）
- excellent：`#a78bfa`（紫）
- profitable：`#4ade80`（绿）
- average：`#94a3b8`（灰）
- bad_release / lose：`#f87171`（红）
- counter_win：`#38bdf8`（蓝）

## 四、接入位置

在每个 `if (state.gamePhase === "xxx") return (...)` 块中，在现有文案 `<div>` 和 `<button onClick={restart}>` 之间插入 `<ShareCard state={state} />` 组件。

组件接收完整 `state`，内部读取所需字段（gamePhase / gameDirection / marketYear / survived / honesty / people / quality / judgment / grit）。

---

## 五、验证点

- [ ] 七种结局画面均出现分享卡
- [ ] 方向未选（gameDirection 为 null）时，「方向 · 年份」行不显示
- [ ] 五维色块条正确反映实际数值
- [ ] 评语按优先级命中正确条件（可在 legendary + 低 honesty 测试）
- [ ] 卡片样式与游戏整体风格一致，暗色背景，不突兀
- [ ] 「再来一局」按钮在卡片之后，不被卡片遮挡

---

## 六、开发提示词

请阅读以下两个文件，按 spec 新增分享卡组件：

- 游戏主文件：`delay-game.jsx`
- 功能规格：`spec-share-card.md`

新增一个 `ShareCard` 组件，接收 `state` prop，内部实现评语逻辑和卡片渲染。在所有七种结局 gamePhase 的返回块中插入该组件，位于文案下方、「再来一局」按钮上方。无需任何外部库，纯 CSS + JSX 实现。
