## 一、核心循环

```
每个回合，玩家有5-7AP，要分配给：

  冲进度        维持士气       维持信任       修根源        紧急应对
  进度+3~5     关怀+10(递减)  向上管理+1    技术检查-15   危机安抚+25(递减)
  士气-4       但不解决根源   但花了AP      qualityDebt   但3AP
  qualDebt+?                               但不推进度

5-7AP不够做所有事。必须选：这个回合我放弃什么？

事件不是发福利，是在已经很紧的盘面上再加一刀。
每个选项都在告诉你：你要付钱，问题是用什么付。
```

## 二、预算加速消耗

```
项目本身就在烧钱，不是"有事才花钱"：

  月1-2：每周 -2    "几个人在工位上敲键盘"
  月3-4：每周 -4    "测试外包 资产结算 服务器扩容"
  月5-6：每周 -6    "全员加班费 发行方预付 验收烧钱"

24周总消耗 = 96（不含任何事件/行动花费）

叙事依据：项目越到后期人越多、测试成本越高、外包结算越密集
体感效果：越到后期越紧，倒计时感极强
```

## 三、士气系统重构

### 3.1 士气分段 × 效率系数

士气不是"归零=死"的开关，是整个系统的传动轴：

| 区间 | 进度行动效率 | 体感 |
|------|------------|------|
| 80+ | ×1.2 | 团队状态很好，进度自然快 |
| 50-79 | ×1.0 | 正常 |
| 35-49 | ×0.8 | 推进效率明显变慢 |
| 20-34 | ×0.6 | 有人在投简历了 |
| <20 | ×0.4 | 摆烂，但还在位置上坐着 |

士气归零不直接解散，但×0.4意味着冲进度只产出不到一半，预算还在烧，时间还在走 → 自然走向失败。

### 3.2 士气自然衰减

士气衰减不是固定值，由项目环境驱动：

```
每周士气自然衰减 = base + qualityDebt压力 + 团队规模负担 + 项目阶段压力

  base:                -1/周
  qualityDebt压力:     floor(qualityDebt / 30)
  团队规模负担:        max(0, teamSize - 3) × 0.5
  项目阶段压力:        floor((month - 1) / 2)

典型场景：
  月1，3人，qualityDebt=0：  -1/周   "创业初期，累但有劲"
  月3，5人，qualityDebt=45： -4/周   "中等规模，质量开始有问题"
  月5，6人，qualityDebt=70： -6.5/周 "后期大团队，做的东西自己也看不下去了"
  月2，2人，qualityDebt=0：  -1/周   "小团队干净代码，士气稳定"
```

博弈：
- 多招人 → 进度快 → 但团队规模↑ → 士气衰减↑ → 必须花更多AP维持士气
- 少招人 → 士气稳 → 但进度慢 → 老板干预 → bossTrust↓
- qualityDebt高 → 士气衰减快 → 产品健康度低 → 关怀行动效果打折（×0.7） → 恶性循环

### 3.3 低士气 → bossTrust传导

月底结算时检查：
- morale < 35 持续整月 → bossTrust -1 "他听说了团队状态"
- morale < 20 持续整月 → bossTrust -2 "他不仅听说了，还收到了离职申请"

不即时触发，月底结算 → 玩家有缓冲期去修复。

### 3.4 高信任 → 士气缓冲

bossTrust ≥ 8 时：士气自然衰减速率 ×0.8

叙事依据："老板信任你，你不用花精力应对干预，团队感受得到这种空间。"

双向关系：
- 低士气 → 信任下降
- 高信任 → 士气衰减减缓
- 形成正/负螺旋，玩家需要同时维护两边

### 3.5 关怀/安抚的递减效应

短期止痛药会失效，不解决根源问题光靠安抚边际效用递减：

```
危机安抚（2AP，基础+25）：
  衰减系数 = 1 / (1 + 0.3 × 使用次数)
  第1次：+25   "终于有人关心我们了"
  第2次：+19   "又来？上次关心完也没变什么"
  第3次：+16   "你在说一样的话"
  第4次：+13   "我们听过这个了"
  第5次：+11   "......"
  下限：+10

团队关怀（1AP，基础+10）：
  递减更温和，下限+5

叙事解释：安抚和关怀是短期刺激，不是结构改善。
根源是qualityDebt、团队规模、项目方向——不解决根源，光靠安抚会失效。
```

策略深度：花AP紧急救士气 vs 花AP修根源问题（技术健康检查、砍掉烂需求）。短期止痛药会失效，但修根源需要时间。

## 四、隐性积分制

### 4.1 五维度定义

| 维度 | 积分来源 | 扣分来源 | 游戏内反馈回路 |
|------|---------|---------|--------------|
| 诚实 Honesty | milestone选B(+2) 选C(+1)；如实汇报(+2) | 选A且未消除水分(-3)；掺水被发现(-2) | 低诚实→进度水分增加→water_reveal概率↑ |
| 人心 People | 获得心腹(+3)；清醒的人选C帮他转岗(+3)；milestone选C用心腹(+1) | 失去心腹(-4)；清洗人员(-2) | 低人心→士气衰减速率额外×1.2~1.5 |
| 品质 Quality | qualityDebt全程<20(+5)；qualityDebt<40(+3)；技术健康检查行动(+1) | 最终qualityDebt每>50一个月(-1) | 已外显：qualityDebt→产品健康度→士气韧性 |
| 判断 Judgment | 方向匹配primary(+5)；匹配secondary(+3)；拒绝跟风坚守方向(+2) | 匹配mocked(-3)；跟风放弃方向(-2) | 已外显：方向匹配→结局门槛 |
| 韧性 Grit | 预算从未紧急融资(+4)；bossTrust从未归零(+3)；从未触发boss_talk(+3) | 紧急融资(-2)；触发boss_talk(-3) | 已外显：预算/信任触底事件 |

### 4.2 积分 → 结局档次

| 积分 | 结局 | 文案方向 |
|------|------|---------|
| 40-50 | 传奇 | "你做到了，而且你知道为什么。" |
| 30-39 | 优秀 | "你做到了。有些人知道代价。" |
| 20-29 | 大赚特赚 | "你做到了。但你不确定值不值得。" |
| 10-19 | 中规中矩+ | "你做到了。但有些东西，活不了。" |
| <10 | 走现有4档结局判定 | — |

"从来不选"某个坏事 → 该维度满分 → 传奇的必要条件
选了一次 → 该维度7-8分 → 优秀可达到，传奇几乎不可能
选了多次 → 该维度3-5分 → 最多大赚特赚

### 4.3 隐性积分不做UI显示

但每个维度有临界提示信号（类似产品健康度），让玩家能感觉但看不到数字：

```
诚实分<3时，milestone结果追加：
  「你在会议上越来越熟练了。熟练到有时候你自己也分不清，
    哪些是真的，哪些是说出来的。」

人心分<3时：
  「走廊里遇到的人越来越少了。你不太确定，
    是他们不在了，还是他们不想跟你说话了。」

品质分<3时：
  「有人在内网发了一篇匿名帖：「这个游戏的核心玩法，
    你自己玩过吗？」没有人回复。帖子第二天被删了。」
```

### 4.4 诚实分的迟滞惩罚回路

不诚实的代价不是立即的，是累积的：

```
月1: 选A（报好看的数字），+5进度，水分小，没事
月2: 选A，诚实分已低，团队也开始"优化数字"，milestone水分自动增加
月3: 选A，水分已经很大，water_reveal爆发，-8进度
     "三个月的泡沫，一个月戳破"

机制：诚实分<5时，每月milestone A选项自动附加额外水分（progress虚高但实际无效进度增加）
逻辑：领导不诚实，下属有样学样，进度泡沫越来越大
```

## 五、事件维度扩展

### 5.1 问题

当前事件选项通常只碰2个显性维度，导致"谁在交换谁"太透明，选择太容易算。

### 5.2 原则

每个选项碰3-4个维度，含1-2个隐性维度：

```
❌ 当前：选项A: progress+5, morale-4          ← 只碰2个，太透明
✅ 改后：选项A: progress+5, morale-4, qualityDebt+8, honesty-1  ← 表面赚进度，暗伤品质和诚实

❌ 当前：选项B: morale+10                     ← 纯收益
✅ 改后：选项B: morale+10, budget-10, bossTrust-1               ← 买到了士气，花了钱，老板不高兴

❌ 当前：选项C: progress-3, budget-5          ← 净亏损，不会选
✅ 改后：选项C: progress-3, qualityDebt-5, people+1             ← 短期痛，长期干净，攒了人心
```

### 5.3 完整维度层

```
显性数值（玩家能看到并管理）：
  progress(0-100) / morale(0-100) / budget(0-100) / bossTrust(0-10)

半显性数值（能感觉到，看不到精确数字）：
  qualityDebt(0-100) / kpiState(L/N/T) / productHealth(4档信号)

隐性积分（完全不可见，影响结局）：
  honesty(0-10) / people(0-10) / quality(0-10) / judgment(0-10) / grit(0-10)
```

事件选项应跨层打击：显性层1-2个 + 半显性层0-1个 + 隐性层0-1个。
