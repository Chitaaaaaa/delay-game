# 背景特殊选项批量生成模板

## 一、10种背景的核心视角

每种背景代表玩家看待问题的独特切入角。
生成选项时，选项的**措辞方式**和**解决路径**必须体现这个视角，而不是通用处理。

| 背景ID | 标签 | 看问题的角度 | 典型行动模式 |
|--------|------|------------|------------|
| `designer` | 策划 | 玩家体验、系统逻辑、功能优先级 | 评估对玩法的影响，从用户视角重构问题 |
| `engineer` | 程序 | 技术可行性、实现成本、代码健康度 | 要求先看技术细节，用数据/代码说话 |
| `artist` | 美术 | 团队氛围、创意质量、视觉/感性判断 | 察觉情绪信号，从美感和文化出发 |
| `pm` | 项管 | 流程、资源分配、deadline、向上汇报 | 把问题转化为可谈判的条件，管理预期 |
| `realestate` | 地产 | 合同条款、甲方关系、利益结构谈判 | 把任何对话变成利益谈判，擅长重签条款 |
| `education` | 教培 | 激励人、结构化培训、人才培养 | 用教练式对话处理人的问题，但低估行业混乱度 |
| `finance` | 金融 | ROI、风险建模、数字优先、成本控制 | 要求量化一切，用财务逻辑看决策 |
| `film` | 影视 | 见过更大的烂摊子，垂直切片验证方法论 | 先做最难的一段证明可行，最小成本试错 |
| `blockchain` | 区块链 | 叙事框架、价值包装、融资话术 | 把问题包装成机会，擅长讲故事但推高预期 |
| `mcn` | MCN/直播 | 流量思维、用户反应、传播效率 | 从受众视角看产品，但对开发实质理解浅 |

## 二、Prompt 模板

将下方模板中的【】替换为具体事件内容，发给 AI 生成选项。

---

```
你是一个游戏文案策划，正在为一款文字模拟游戏《完蛋！我被延期包围了》编写事件选项。

游戏背景：玩家扮演一个游戏制作人，需要在有限预算、时间和团队内完成一款游戏。
核心数值：progress（进度0-100）、morale（士气0-100）、budget（预算0-130）、bossTrust（老板信任0-10）、qualityDebt（技术债0-10）、directionClarity（方向感0-100）

---

【事件名称】：___
【事件情境】：___
【事件对话】：___
【现有普通选项】：
1. 选项文本 → 效果：progress±N, morale±N, budget±N, bossTrust±N
2. 选项文本 → 效果：...
3. 选项文本 → 效果：...

---

现在请为以下10种玩家背景，各生成1个额外的特殊选项。

每种背景的选项要体现该背景玩家的独特视角和行动方式（见下方视角说明），
不能只是换个说法的普通选项，必须有真实的「只有这个背景才能这么做」的感觉。

【背景视角说明】：
- 策划(designer)：从玩家体验、系统逻辑视角切入，评估对玩法的影响
- 程序(engineer)：要求看技术细节，用代码/数据说话，评估实现成本
- 美术(artist)：察觉情绪信号，从团队氛围和感性判断出发
- 项管(pm)：把问题转化为可谈判的条件，管理资源和预期
- 地产(realestate)：把对话变成利益结构谈判，擅长重签条款
- 教培(education)：用教练式对话处理人的问题，激励为主
- 金融(finance)：量化一切，用ROI和风险模型看决策
- 影视(film)：先做最小验证，最低成本试错，见过更大的烂摊子
- 区块链(blockchain)：把问题包装成机会叙事，擅长讲故事
- MCN(mcn)：从用户/流量视角看，但对开发实质理解浅

【输出格式】（严格按此格式，10个，不要多余解释）：

designer:
  text: "选项按钮文本（10-25字，第一人称）"
  effects: { progress: 0, morale: 0, budget: 0, bossTrust: 0 }  // 只写有变化的字段
  result: "玩家看到的结果描述（40-80字，叙述性，有画面感）"

engineer:
  text: "..."
  effects: { ... }
  result: "..."

artist:
  ...

pm:
  ...

realestate:
  ...

education:
  ...

finance:
  ...

film:
  ...

blockchain:
  ...

mcn:
  ...

---

【效果设计原则】：
1. 特殊选项的效果通常比普通选项「更精准」或「有独特的副作用」，不是单纯更好
2. 每个背景的选项应该有1-2个有变化的数值字段，最多3个
3. 效果范围参考：progress ±1~8，morale ±3~10，budget ±5~15，bossTrust ±1~2，qualityDebt ±1~5
4. 有些背景擅长某类问题但有盲点（如blockchain擅长budget但推高qualityDebt）
5. result文字要有代入感，体现该背景玩家的思维方式
```

## 三、使用示例

以「屎山」意外事件为例，填入模板：

```
【事件名称】：屎山
【事件情境】：[角色名] 来找你：
【事件对话】：「我需要跟你说一件事。新来的同学看了一下这块代码……他说，他不知道从哪里开始改。」
【现有普通选项】：
1. 「专项整治，这周先停新功能」→ progress-10, qualityDebt-3
2. 「边做边改，能跑就行」→ progress-3, qualityDebt-1
3. 「新人适应一下就好了」→ qualityDebt+1, morale-3
```

期望输出示例（engineer）：
```
engineer:
  text: "让我看看代码，告诉你真实的重构范围"
  effects: { progress: -4, qualityDebt: -2 }
  result: "你花了半天看了关键模块。问题比想象的小，但也比他说的复杂。你拆出了必须改的部分和可以先不动的部分——至少有个边界了。"
```

## 四、批量使用说明

1. 每次填入一个事件，发给 AI
2. 拿到10个选项后，逐条检查：文本是否有「只有这背景才能这么做」的感觉？效果是否有独特性？
3. 调整后整理进 `getIndustryBackgroundChoices` 函数（同时支持 playerBackground 和 industryBackground 查找）
4. 优先处理高频事件（每局必触发的事件先做）
