# Patch 36：现有事件迁移到角色系统

> 前置：Patch 01-35 已完成
> 目标：将现有事件归属到角色系统，分三类处理：
>       A) 由 NPC 人格带来（挂载到角色）
>       B) 由上级/外部带来（Tom/老板，不在8人格内）
>       C) 保持为无归因的世界事件

---

## 一、归属分类

### A 类：归属到 NPC 人格

| 现有事件 | 归属人格 | 说明 |
|---------|---------|------|
| lucid_p1 / lucid_p2 | 完美主义 | 主程是完美主义者，敢说真话 |
| zombie_reveal | 幻想家 | 幻想家制造了虚假进度，或被完美主义揭穿 |
| stock_trap | —（减员类意外，见 patch-39） | 触发来自任意角色 |
| team_shortage | 铁头功 | 铁头功最早发现并上报人手问题 |

### B 类：由 Tom / 老板带来（上级，独立于8人格）

Tom 是游戏中的固定上级角色，同时承担两个职能：
- **管理层职能**：代表老板/公司的意志施压
- **HR 职能**：主导招募、人员调动、headcount 审批

| 现有事件 | 来源 | Tom 的职能角色 |
|---------|------|--------------|
| boss_talk | Tom | 管理层 |
| kpi_review | Tom / 老板 | 管理层 |
| manpower | Tom | HR（审批扩编）|
| brooks_law | Tom | HR（推动紧急招募）|
| hire_reveal | Tom / 系统 | HR（揭晓招募结果）|
| 减员（意外事件）| Tom 知情并通知 | HR（离职备案）|

recruit 类行动（campus/social）由玩家主动发起，Tom 不介入，但 headcount 审批隐性需要 bossTrust 支撑（bossTrust 低时招募成本上升，见后续 patch）。

### C 类：无归因的世界事件（保持原样）

| 现有事件 | 说明 |
|---------|------|
| market_trend | 市场变化，外部不可控 |
| thunder | 老板情绪爆发，不归咎于某个8人格角色 |
| milestone 系列 | 节点事件，系统触发 |
| hire_reveal | 系统结果揭晓 |

## 二、迁移实现方式

A 类事件挂载到对应人格后，触发逻辑变为：

```
原：系统随机选取事件池中的事件
新：当该人格角色「本周出现」时，从其事件池中选取一个事件
```

事件的 `id` / `choices` / `effects` 内容不变；
新增 `triggeredBy: charId` 字段，用于关系值结算。

### 2.1 zombie_reveal 的特殊处理

zombie_reveal 原本是剧情里程碑事件，不完全适合归到幻想家的常规出现频率。
处理方式：保留为一次性触发事件，但触发时在叙事上关联当前局内的「幻想家」角色：

```js
// zombie_reveal 触发后，叙事文案中插入角色名：
`「${幻想家角色名}推动的那个『60%完成度』……你深入了解后才发现：」`
```

关系值结算：zombie_reveal 揭穿后，幻想家关系值 -1。

---

## 三、Tom 作为固定角色

Tom 是游戏中唯一的「固定命名上级」，不参与8人格的随机生成，也不在角色池中。
他的出现由现有触发逻辑控制（boss_talk、里程碑、kpi_review 等），不受关系值系统影响。

---

## 四、不改的内容

- 所有现有事件的 choices / effects 数据不变
- B 类和 C 类事件的触发逻辑不变
- boss_talk 的信任恢复机制（patch-32）不变
