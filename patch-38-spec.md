# Patch 38：伞兵完整弧线（修订版）

> 前置：Patch 01-37 已完成
> 依赖：patch-42（角色圣经）
> 目标：实现伞兵的完整弧线——入场、存在期、离场
>       伞兵 = Tom 刻意安插的心腹嫡系，用来制衡、架空、最终替换玩家

---

## 一、设计前提

- 伞兵只在 中厂/大厂 激活（小厂架构扁平，Tom 没有多余岗位安插心腹）
- 触发条件：bossTrust < 3 持续 N 周（大厂 N=2，中厂 N=3）
  — 这是 Tom「等待破绽」的机制：玩家信任度崩盘 = Tom 有了出手的正当理由
- 伞兵的表面话术是「协助稳定项目」
- 伞兵的真实目标：分割管理权 → 渗透团队 → 顶替玩家
- 玩家无法直接对抗 Tom 或开除伞兵，只能通过恢复 bossTrust 让 Tom 失去出手动机

---

## 二、大/中厂伞兵身份差异

| 规模 | 伞兵身份 | 话语权 | 来源 |
|------|---------|-------|------|
| 中厂 | Tom 从其他业务线抽调的直属下属、老部下 | 中等，以「协助、协同、监督」为名慢慢渗透 | Tom 的历史人脉 |
| 大厂 | Tom 依托集团人脉调来的总部派驻督查/专项负责人 | 极高，背靠Tom+集团双重背书，几乎可以制衡全部决策 | 集团层面 |

## 三、state 新增字段

```js
paratrooper: {
  active: false,
  name: "",               // 开局生成，入场时揭晓
  scale: "",              // "mid" | "large"，决定叙事变体
  weeksActive: 0,
  influence: 0,           // 0~5，Tom的控制权渗透程度
  departed: false,
  departureReason: null,  // "player_won" | "player_lost"
},
lowTrustStreak: 0,
highTrustStreak: 0,
```

## 四、入场事件

触发时机：`lowTrustStreak` 达到阈值的那一周，在 planning 阶段开始前强制插入。

**入场叙事（单一确认选项，无法拒绝）：**

```
Tom 发来一条消息：

「项目风险偏高，进度节奏也不够稳定。
我安排[伞兵名]协同跟进，辅助你稳定节奏。
他经验丰富，配合就行。」

[收到]
```

> 注：Tom 不说「你做得不好」，不说「我不信任你」。
> 措辞永远是「项目风险」「节奏稳定」——他在说项目，不在说你。
> 但所有人都知道他在说什么。

**中厂变体补充叙事（紧接确认后出现）：**
```
你搜了一下[伞兵名]的名字，他在公司内网上挂的title是「业务发展总监」。
游戏研发经验：无。
汇报关系：Tom。
```

**大厂变体补充叙事：**
```
内网通知同步下发：集团总部委派[伞兵名]担任本项目专项负责人，
全面协助项目管理工作，直接向Tom汇报。
你注意到这份通知，抄送了老板。
```

效果：`paratrooper.active = true`，`paratrooper.name / scale` 赋值。

## 五、存在期机制

### 5.1 每周被动影响

```js
if (paratrooper.active && !paratrooper.departed) {
  paratrooper.weeksActive += 1;

  // influence 与 bossTrust 反向联动
  if (bossTrust < 4) {
    paratrooper.influence = Math.min(5, paratrooper.influence + 1);
  } else {
    paratrooper.influence = Math.max(0, paratrooper.influence - 1);
  }
}
```

### 5.2 伞兵事件（每2周触发一次）

玩家面对的选项通常没有「好的选择」，只有「代价大小」：

**事件一：「你的意见」**
```
情境：Tom 在周会上问你怎么看[伞兵名]提出的方案。

选项A：支持他
  效果：bossTrust+1, paratrooper.influence+1
  结果：「Tom 点了点头。[伞兵名]看了你一眼，没有表情。」

选项B：提出不同看法
  效果：bossTrust-1
  结果：「Tom 没有接话。会议结束后你收到一条消息：
        『下次重要场合，建议先同步。』
        发消息的是[伞兵名]。」
```

**事件二：「分工调整」**
```
情境：[伞兵名]发来一份邮件，提议「优化分工」——
      他来「统筹团队协调和对外沟通」，你专注「内容产出和研发执行」。

选项A：同意
  效果：morale-3, paratrooper.influence+1
  结果：「从这周开始，对外的邮件上多了一个抄送名字。
        他的。」

选项B：拒绝，保持现有分工
  效果：bossTrust-1
  结果：「他回复了两个字：『好的』。
        你不确定这是接受，还是记下来了。」
```

**事件三：「他的判断」**
```
情境：你的一个决策，[伞兵名]在群里发言表示「有些疑虑」。

选项A：解释并坚持
  效果：progress-2, 有机会bossTrust+1（取决于解释质量）
  结果：「群里安静了十分钟，然后有人说『制作人说得有道理』。
        你不确定[伞兵名]是不是记住了这件事。」

选项B：沉默，让它过去
  效果：paratrooper.influence+1, morale-2
  结果：「没有人为你发言。那条消息悬在群里，
        没有人明确反驳它，但也没有人完全同意。」
```

**事件四：「背后的话」**
```
情境：一个信任你的同事悄悄来找你——

「[伞兵名]昨天在Tom办公室呆了两个小时。
我不知道他们说了什么，但出来的时候，Tom跟他说『先这样，看看』。」

选项A：正面应对，找Tom直接问
  效果：bossTrust±0（结果取决于当前bossTrust高低）
  结果：若bossTrust≥5：「Tom说：『正常沟通，你多虑了。』你不确定他说的是不是真的。」
        若bossTrust<5：「Tom说：『有什么问题吗？』你无话可说。」

选项B：先忍着，等时机
  效果：morale-2，埋下伏笔
  结果：「你什么都没说。这件事就这么过去了。
        你不知道它会不会以另一种形式回来。」
```

### 5.3 influence 的叙事可见性

influence 不显示数字，通过行为变化暗示：

| influence | 叙事信号 |
|-----------|---------|
| 1–2 | 「Tom 最近开会，有时会先看他一眼再做决定。」|
| 3–4 | 「你发现有几封需要你签字的邮件，现在先到了[伞兵名]那里。」|
| 5 | 触发「权力转移」——进入伞兵失败结局 |

## 六、离场机制

### 6.1 玩家赢（bossTrust 回升）

```js
if (paratrooper.active && bossTrust >= 7) {
  highTrustStreak += 1;
} else {
  highTrustStreak = 0;
}
if (highTrustStreak >= 3) triggerParatrooperDeparture("player_won");
```

**离场叙事（Tom腔）：**

```
Tom 发来消息：

「跨部门人力调整，[伞兵名]下周起回归原业务线。
后续项目由你独立全权负责。」

[收到]
```

补充叙事（玩家内心）：
```
你盯着这条消息看了一会儿。

他没有说你做得好，也没有说他错了。
他只是在告诉你一个人员调动。

你把手机放下，继续工作。
```

效果：`paratrooper.departed = true`，`departureReason = "player_won"`，bossTrust+1。

### 6.2 玩家输（influence 达到 5）

触发专属失败结局（`ending_paratrooper`）：

```
Tom 把你约到会议室。

「项目目前的状态，你应该清楚。
[伞兵名]对这个项目已经比较熟悉，
我们评估之后，认为接下来由他来负责，更适合。」

他停顿了一下。

「试用期的事，HR会跟你确认。」

你走出会议室。
楼道里有人叫了你一声。你没有回头。
```

## 七、伞兵关系值的反向逻辑

- relationship 高（伞兵「喜欢」你）→ 危险信号，他在接近你以获取信息和减少阻力
- relationship 低（公开对立）→ 说明玩家在有效抵抗，但会加速bossTrust损耗

UI 不显示 relationship 数字，只通过事件叙事体现。
