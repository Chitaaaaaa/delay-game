# 产品健康度信号 spec

## 定位

**产品健康度**是一个副属性信号，嵌套在士气面板的折叠区域内。

它回答的问题是：团队私下对这个游戏有多少信心？

- 老板不知道这个数字（他看的是进度汇报和milestone演示）
- 团队知道（每天在做，好不好心里有数）
- 玩家通过士气面板可以看到信号，但看不到具体数字

这个设计让产品健康度进入**生存博弈**（影响士气韧性），而不只是在结局画面才露面。

---

## 输入来源

| 来源 | 权重 | 说明 |
|------|------|------|
| qualityDebt | 主要 | 技术债/设计债，范围 0-100 |
| 方向匹配度 | 次要 | gameDirection 与 marketYear 热门品类的吻合程度 |

方向匹配度惩罚计算：
- 方向吻合：0 惩罚
- 方向未选 / 中性：+15 惩罚
- 方向明显不吻合：+30 惩罚

综合分：
```
productHealthScore = 100 - qualityDebt × 0.7 - directionMismatchPenalty × 0.3
```

---

## 信号状态（4档）

| 分数区间 | 信号 | 展示文案 |
|---------|------|---------|
| ≥ 75 | ↑↑ | 「团队对这个游戏有信心」 |
| 45–74 | → | 「大家在正常推进」 |
| 20–44 | ↓ | 「团队开始质疑这个游戏的方向」 |
| < 20 | ↓↓ | 「团队私下都知道这个游戏有问题」 |

---

## 机械效果

产品健康度通过影响**士气的韧性**作用于核心循环，而不是直接修改士气数值。

| 状态 | 效果 |
|------|------|
| ↑↑ | 士气回复行动（关怀/安抚）效果 ×1.15 |
| → | 无修正 |
| ↓ | 士气回复行动效果 ×0.85；负面事件的士气伤害 ×1.1 |
| ↓↓ | 士气回复行动效果 ×0.7；负面事件的士气伤害 ×1.2；每周自然流失士气 -2 |

**玩家的感知**：不是"qualityDebt=43所以效果打折"，而是"最近关怀行动好像不太管用，而且事件总是特别伤士气"。打开士气折叠菜单才会看到产品健康度信号在下滑。

---

## UI 规格

**位置**：士气主数值条下方，折叠区域（默认折叠）

**折叠后显示**（一行）：
```
[信号emoji] 产品健康度  [状态文案]
```

例：
```
↓  产品健康度  团队开始质疑这个游戏的方向
```

**不显示**：
- 不显示 productHealthScore 具体数字
- 不显示 qualityDebt 的值
- 不显示各维度权重或计算公式
